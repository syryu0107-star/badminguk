# 배드민국 콜드스타트 전략 — 기존 대회 데이터를 어떻게 채울까

## 0. 결론 먼저 (TL;DR)

- **3대 난제는 사실상 하나의 문제다: "검증된 실력 라벨의 부재".** 데이터가 없어서 초기 MMR을 못 정하고, 검증이 없어서 샌드배깅을 못 막고, 검증된 실력 네트워크가 없어서 주최자가 안 온다. 셋 다 뿌리가 같다.
- **정공법은 "앱 깔아라" 마케팅이 아니라 "대회 1건 운영을 무료로 대행 → 참가자 전원 기록을 시드로 회수"다.** 스포넷·위꾹이 실제로 걸어온 길이 이것이고, B2G/B2클럽이 개인 마케팅보다 압도적으로 효율적이다.
- **초기 MMR은 정확히 맞출 필요가 없다.** Glicko-2식으로 "불확실성을 크게 두고 5~10경기 만에 강제 수렴"시키는 엔진이 정답. 초기값이 틀려도 자기교정된다.
- **지금 배드민국이 만들 것: (1) 주최자용 CSV 대회결과 임포터, (2) 급수→MMR+RD 밴드 매핑, (3) provisional 큰-K 수렴 엔진, (4) 신뢰도 뱃지, (5) 자기신고+증빙 온보딩.**

## 1. 핵심 진단 — 진짜 문제는 데이터가 아니라 "신뢰"다

사용자가 제기한 3난제를 다시 보면:

| 표면 문제 | 실제 뿌리 |
|---|---|
| (1) 초기 MMR 근거 없음 | 실력을 증명할 **검증된 라벨**이 없음 |
| (2) 고급수의 하향 신고(샌드배깅) | 신고를 **검증·반박할 데이터**가 없음 |
| (3) 주최자가 안 옴 | 주최자에게 줄 **네트워크 가치**가 없음 |

세 문제는 독립적이지 않다. **하나의 자산 — "협회/기존 플랫폼에 이미 존재하는 공인 급수·입상 기록" — 을 배드민국 좌표계로 이식하면 셋이 동시에 풀린다.**

- 급수를 이식하면 → 초기 MMR 앵커가 생긴다 (난제 1 해결).
- 이식된 기록은 하향 신고를 즉시 반박한다 → 샌드배깅 억제 (난제 2 해결).
- 이식 대상이 곧 "주최자가 이미 운영한 과거 대회"이므로, 임포트 자체가 주최자를 끌어오는 유인이 된다 (난제 3 해결).

**따라서 진짜 문제는 "데이터를 새로 생성"하는 것이 아니라 "이미 존재하는 데이터를 합법·정확하게 이식"하는 것이다.** 배드민국은 데이터를 0에서 만드는 게 아니라, 흩어진 검증 자산을 자기 좌표계로 회수하는 게임을 해야 한다.

**냉정한 현실 인식**: 급수라는 "공인 실력 라벨"이 한국에 이미 전국 단위로 존재한다는 건 큰 행운이자 동시에 함정이다. 행운인 이유는 앵커가 공짜로 있다는 것. 함정인 이유는 **그 라벨을 이미 스포넷·위꾹·배프가 쥐고 있어서, 배드민국이 "또 하나의 급수 앱"이면 존재 이유가 없다**는 것. 차별화는 규칙(누구나 아는 Glicko·급수 게이팅)이 아니라 **"기록 확보의 범위·정확도·합법성"**에서만 나온다.

## 2. 위꾹/스포넷은 실제로 어떻게 시작했나 (밝혀진 것 / 추정 구분)

### 리서치로 확인된 사실

- **위꾹라이브 = 명백한 B2G(협회 영업) 부트스트랩.** 2022년 6월 시범서비스, 2024년 12월 기준 **서울시 25개구 중 22개구 배드민턴협회와 계약 완료.** 주 고객이 서울/경기/인천 등 공공 배드민턴협회. 개별 동호인이 아니라 **대회 주최자(협회)를 계약으로 먼저 잡아** 닭-달걀을 주최자 쪽부터 깼다. (출처: ohmycompany.com/invest/379)
- **스포넷 = 대회 운영 도구가 데이터를 부산물로 축적하는 구조.** 협회 대회의 실시간 대진·점수 운영권을 잡고, 대회가 열릴 때마다 참가 기록이 쌓여 급수 명단·국가랭킹으로 이어진다. 데이터가 "서비스(운영)의 부산물"로 축적. (출처: play.google.com, sponet.co.kr)
- **배프 = 출전기록 기반 급수 게이팅을 제품 규칙으로 구현.** 과거 참가·입상 기록으로 신청 가능 급수가 자동 결정되고, 복식 파트너 중 높은 급수로 팀 급수가 승격된다. (출처: badmintonfriends.co.kr)

### 추정 영역 (근거로 인용하지 말 것)

- 스포넷의 **구체적 랭킹 산정 알고리즘·데이터 수집 방식은 비공개.** 내부 방식은 추측.
- "연 100+ 대회" 같은 수치는 **벤더 소개 문구**라 신뢰 근거로 삼지 말 것.
- 협회가 외부 개발자용 **공개 API를 제공하는지는 미확인.** 배드민국이 자동 연동에 로드맵을 의존시키면 위험.

### 배드민국이 가져갈 교훈

**"동호인에게 앱을 팔지 마라. 주최자에게 무료 운영 도구를 주고 결과 데이터를 회수하라."** 이게 두 선발주자의 공통 DNA다. 단, **위꾹이 이미 서울 대부분 구 협회를 선점**했으므로 정면 경쟁은 불리. 위꾹이 약한 곳 — **소규모 클럽 자체대회, 특정 지방, 비협회 동호회 리그** — 으로 우회 진입해야 한다.

## 3. 초기 MMR/급수 부트스트랩 구체안

### 3-1. 대원칙: 정확한 초기값을 포기하고, 빠른 수렴을 산다

레이팅 업계 공통 정답: **초기값을 맞추려 하지 말고, 불확실성을 크게 둔 뒤 몇 경기 만에 강제 수렴시킨다.**

- Glicko-2: 신규는 RD(불확실성)=350으로 시작, RD가 크면 경기당 점수 이동폭이 커져 진짜 실력으로 빠르게 당겨진다. 15~20경기 후 provisional 해제, 활동 선수는 RD 30~80로 안착.
- USCF: 첫 25경기 provisional, K=800/유효경기수로 초반 변동폭을 키움.
- TrueSkill: μ=25, σ=8.33에서 σ를 성과가 예측과 맞을수록 축소. (단 MS 특허·라이선스 확인 필요 → **배드민국은 Glicko-2 권장**)

배드민국은 이미 급수 3축(구/시/전국 × 단/복)과 MMR RPC를 갖고 있으므로, **그 위에 "RD(또는 신뢰도) 컬럼 하나 + provisional 큰-K 로직"만 얹으면 된다.**

### 3-2. 자기신고를 "감"이 아니라 "예측 변수"로 받는다

NTRP/UTR의 교훈: **본인 급수를 직접 묻지 마라. 검증 가능한 사실을 물어라.** 이게 본인 직감보다 실력을 훨씬 잘 예측한다.

온보딩 설문 (배드민국판):

```
① 선수부 출신 여부 (초/중/고/대/실업)   → 자강권 강력 신호
② 구력 (레슨 시작 시점)                  → 급수 하한
③ 소속 클럽                             → 클럽 운영진 교차검증 대상
④ 최근 출전 대회 + 출전 조 + 성적        → 입상이력 역추정 입력
⑤ 주 운동 횟수                          → 보조 신호
⑥ 현재 협회 급수 (선택) + 증빙          → 앵커 (증빙 유무로 가중치 차등)
```

**이 답을 "점 추정치"가 아니라 "범위"로 변환한다.** 예: "선수부 무 + 구력 2년 + 지역 C조 출전" → **초기 MMR = C~B 밴드의 중앙값, RD = 밴드 폭.** 자기신고의 불확실성이 그대로 수학적 RD로 흡수된다.

### 3-3. 급수 → MMR 밴드 매핑 (앵커 좌표계)

무근거로 1500을 주지 말고, 협회 공인 급수를 밴드로 앵커:

| 급수 | 초기 MMR 중앙 (예시) | 초기 RD |
|---|---|---|
| 자강 | 높음 (상단) | 좁게 (검증된 라벨) |
| 준자강 | ↓ | 중 |
| A | ↓ | 중 |
| B | 중상 | 넓게 |
| C | 중 | 넓게 |
| D | 중하 | 넓게 |
| 초심 | 하단 (바닥값) | 최대 |

핵심 규칙 3가지:
1. **바닥값(floor)을 둔다.** 초보가 무한 패배 루프에 빠지지 않게 (FIDE 1400, USATT 200 사례).
2. **이식 초기엔 외부 급수를 "순위(ordinal)"로만 신뢰하고 절대 점수는 자체 경기로 재보정.** 외부 급수의 절대 스케일이 배드민국 MMR과 안 맞으면 앵커 전체가 편향된다.
3. **급수 기준은 지역협회마다 다르므로** 밴드를 넓게 잡아 지역 편차를 흡수. 협회 미가입 순수 동호인은 급수가 없으니 설문 기반 provisional로만.

### 3-4. 첫 경기 역산 앵커

USATT/UTR/FIDE 공통: **첫 경기 1건이면 상대 레이팅 + 스코어 마진으로 잠정 레이팅을 즉시 역추정.**

- 배드민국 신규 선수의 첫 경기 상대를 **"이식 데이터가 붙은 기존 급수 선수"**로 매칭.
- 21-x 게임 차(스코어 마진)로 초기 MMR 역산.
- 이식한 급수 선수가 바로 이 **앵커**가 된다 → 콜드스타트를 "기존 급수를 앵커 좌표계로 이식"해 정면 돌파.

### 3-5. 무활동 재보정 (복귀 샌드배깅 자동 흡수)

Glicko의 무활동 RD 증가: `RD_new = √(RD² + c²)` (350에서 클리핑). 오래 쉰 선수는 자동으로 다시 불확실 상태가 되어 provisional 큰-K가 재적용된다. **"오래 쉬다 복귀한 고수"의 하향 재진입도 같은 메커니즘으로 잡힌다.**

## 4. 샌드배깅 초기 방지 — 무엇을 배드민국에 넣나

핵심 철학: **"신고를 막지 말고, 경기 결과가 신고를 빠르게 짓밟게 하라."** (NTRP는 신고를 웃도는 성과 3경기면 자동 DQ.)

배드민국에 넣을 것을 **비용 대비 효과 순으로**:

### 넣는다 (필수, 지금)

1. **자기신고 ↔ 입상이력 교차검증 플래그.** 신고 급수가 역추정 급수보다 2급 이상 낮으면 자동 검증 큐 회부. 규칙 기반이라 데이터 적어도 즉시 작동.
2. **provisional 큰-K + 첫경기 급상향.** 저급수 신고자가 첫 몇 경기를 큰 점수차로 압승하면 MMR이 급등해 위장이 저절로 노출. 정착 후 K 축소로 소프트 상대 파밍 억제.
3. **단방향 래칫(하향 불가).** 한번 부여된 MMR/급수는 아래로 못 감. 실제 경기로만 상승/유지. (협회의 "승급 후 1개월 뒤 새 급수 강제" 원리와 동일.)
4. **분단위 급수 상한(division cap).** 대회 등록 시 앵커 MMR이 상한 초과면 하위 조 등록 자체를 시스템이 차단. 자가선택으로 낮은 조에 들어가는 통로를 막으면 샌드배깅 유인이 사라진다. → 주최자에겐 "MMR 자동 시드/조편성" 기능으로 제공(유치 인센티브).
5. **1인 1계정 휴대폰 본인인증.** "새 계정으로 숫자 리셋" 트릭 원천 차단. **단 초기 전환율을 위해 단계적 게이팅** — 가입은 쉽게, 대회 출전/랭킹 반영 시점에 인증.

### 넣는다 (증빙, 선택+보상형)

6. **캡처 인증 with 신뢰도 가중.** 무증빙 < 스포넷 전적 캡처 < 협회 경기인 등록증 순으로 초기 MMR 신뢰도 차등. **증빙 강제는 이탈을 부르니 "선택+보상(인증 시 배지/신뢰도↑)"으로.**

### 넣는다 (사람 개입, 소규모 단계에 오히려 실효적)

7. **페어플레이 검토 큐: 비정상 압승 자동 플래그 + 커뮤니티 신고 버튼 + 운영자 재분류 권한.** 신생 단계에선 화려한 이상치 탐지보다 "주최자 확인 + 커뮤니티 신고"가 저비용·고실효. 한국 동호인은 실명·소속 클럽이 노출돼 평판 압력이 자정에 유리(단 미검증 가설). **신고만으로 제재 금지 — 경기 데이터 근거 필수(무고 방지).**

### 출처별 가중치 (DUPR식, 관통 원칙)

**주최자 확정 > 심판/클럽 인증 > 상대 상호확인 > 본인 단독입력.** 셀프 기록만 쌓아 낮은 숫자를 유지하는 트릭을 무력화. 단 신생 플랫폼은 초기에 주최자 확정 데이터가 적어 저가중 데이터에 의존할 수밖에 없는 딜레마 — 이건 4장이 아니라 5장(주최자 온보딩)으로 풀어야 한다.

### 넣지 않는다 (초기엔)

- 실력 향상 계수 비공개·정교한 통계 이상치 탐지 → 표본 적을 때 오탐 높음. 표본 쌓인 뒤로 연기.

## 5. 닭-달걀 돌파 순서

### 원칙: 어려운 쪽(공급=주최자)을 손으로 먼저 채운다

OpenTable(식당 먼저), Airbnb(뉴욕에 상주), Uber(드라이버 먼저)의 공통 교훈. **배드민국에서 "어려운 쪽"은 선수가 아니라 주최자다.**

### 최소 공략 단위 = "동호회/클럽 운영진" (양면을 일면으로 축소)

**타깃 1순위는 개인 선수도, 대형 대회사도 아닌 "클럽 운영진".** 이유:
- 정기 내부 리그를 열어 **대회 공급**을 만든다.
- 소속 회원 수십 명을 **선수 수요**로 데려온다.
- 회원 실력을 이미 알아 **초기 급수 시드의 신뢰 있는 정보원**이 된다.

**클럽 1개 온보딩 = 공급·수요·초기데이터 3난제를 한 번에 공략하는 최소 단위.** (eBay·Etsy가 "사는 쪽=파는 쪽" 유저로 양면을 일면으로 줄인 것과 동일.)

리스크: 운영진은 무보수·바빠서 학습 동기 약함 → **온보딩을 거의 대행 수준으로 낮춰야.** 운영진 1인 의존은 단일점 실패(그 사람 이탈 시 클럽 전체 이탈).

### 실행 순서

```
1단계: 운영자가 물리적으로 닿는 "한 지역 + 활발한 클럽 2~3개"에 집중
        → 전국·전종목 동시 공략 금지 (넓게 얕으면 어디서도 신뢰 임계 미달로 죽음)

2단계: 그 클럽의 "과거 대회 결과 CSV 임포트" 대행 → 참가자 전원 프로필+초기 MMR 자동 생성
        (주최자 1명 업로드 = 수십~수백 명 온보딩. 닭-달걀의 지렛대)

3단계: 그 클럽의 다음 내부 리그를 배드민국으로 운영 (대진표·실시간 점수 무료 대행)
        → 자체 경기 데이터가 이식 데이터를 재보정하기 시작

4단계: "이 지역에선 배드민국 급수가 표준"인 국소 유동성 확보 후 옆 지역/클럽으로 복제
        → 복제 가능한 온보딩 플레이북(주최자 영업→과거데이터 임포트→선수 초대) 표준화

부트스트랩(수요 못 기다릴 때): 운영자가 직접 소규모 오프라인 대회(32~64명) 1건 개최
        → 자기 툴 엔드투엔드 검증 + 첫 시드 데이터 정당 생성 + "배드민국으로 굴러간 대회" 레퍼런스
        (단 이건 점화용이지 사업모델 아님. 확장 불가)
```

### Single-player 가치 (네트워크 0명이어도 쓸 이유)

**"Come for the tool, stay for the network."**

- **주최자 쪽 단독가치**: 자동 대진표 + 온라인 접수 + 참가비 결제 + 실시간 스코어. 선수 네트워크가 0이어도 **접수·정산 노동 절감**이 즉각적 가치. 주최자가 쓸 이유가 생긴다.
- **선수 쪽 단독가치**: "내 배드민턴 기록장". 상대가 0명이어도 자기 경기·파트너·구장·급수 이력을 혼자 기록/시각화(승률, 요일별 폼, 상대별 전적, 예쁜 전적 카드). 이 개인 기록이 모이면 자동으로 MMR·매칭 네트워크의 원료가 된다.

리스크: 대회운영 SaaS(Playinga·PlayPass 등)는 이미 존재 → **툴만으로는 차별화 약함.** "한국 급수/MMR 데이터 네트워크"라는 국지적 해자가 있어야 무료 툴이 의미를 가진다. 툴 품질이 스포넷/위꾹보다 명확히 낫지 않으면 전환 안 됨. 기록 입력 마찰이 크므로 단독가치가 "기록 노동"을 넘어설 만큼 즉각적(자동 급수 추정, 공유 카드)이어야 이탈 안 함.

### "기존 대회를 배드민국에 옮기면 주최자가 얻는 것"

1. 접수·참가비 결제·대진표·실시간 순위를 무료 자동화 (노동 절감)
2. 과거 대회 결과 업로드 시 참가자 전원에게 프로필/급수/MMR 자동 부여 (내 대회의 자산화)
3. MMR 기준 자동 시드/조편성 → 샌드배깅 없는 공정한 조 편성 (분쟁 감소)
4. 다음 대회 참가자 풀 확보 (내 대회가 배드민국 네트워크에 노출)

## 6. 기존 대회 데이터 반영 실무 + 합법성 경계

### 6-1. 합법성 경계 (먼저 못 박는다 — 순서 중요)

**자동 크롤링을 1차 전략에서 배제한다.**

- 야놀자 v. 여기어때 (대법 2021도1533): 경쟁사 공개정보 API 크롤링에 대해 **형사(정보통신망법·저작권법·업무방해) 전부 무죄.** 그러나 **같은 사안 민사에서 부정경쟁방지법 (파)목("타인의 상당한 투자·노력으로 만든 성과의 무단사용")으로 손해배상 인정.** 다른 사건에선 10억원대 배상 사례. 개정법상 고의면 손해액 **최대 5배 징벌배상.**
- **"형사 무죄 = 크롤링 합법"은 치명적 오독.** 민사 성과도용·DB제작자권리는 별개로 살아있다.
- 대회 결과엔 **실명+소속클럽+연령대**가 흔히 노출 → **개인정보보호법 동의·최소수집** 대상. 제3자 선수 기록을 임의 저장·게시하면 위법 소지.

**따라서 합법성 우선순위:**

```
안전 ────────────────────────────────────► 위험
주최자 CSV 업로드   본인 셀프 캡처 업로드   협회 공식 제휴/API   경쟁사 자동 크롤링
(권리자 제공)      (정보주체 본인 제공)    (동의 기반)         (민사 리스크 치명적)
   ✅ 주축            ✅ 개인 경로          🔜 2단계          ❌ 배제
```

### 6-2. 주최자 CSV/엑셀 일괄 임포트 (콜드스타트 핵심 기능)

가장 안전+정확. 데이터 권리자(주최자)가 스스로 제공하므로 저작권·개인정보 해소.

임포터 필수 사양:
- **표준 CSV 템플릿**: 선수명 / 성별 / 급수 / 종목(단·복) / 순위 / 상대 / 대회규모(팀수)
- **인코딩 자동감지** (EUC-KR ↔ UTF-8, 한글 깨짐 방지)
- **미리보기 + 중복 병합** (동명이인·기존 프로필 매칭)
- **타인 PII 동의 체크** (참가자 명단 게시 전 마스킹/동의)
- **이미지 대진표 OCR / 수기입력 보조** (원본 CSV 폐기했거나 수기 대진표 사진만 있는 경우 대비 — 이거 없으면 실효성 절반)

**주최자 1명의 업로드가 참가자 수십~수백 명을 한 번에 온보딩.** 이게 닭-달걀의 진짜 지렛대.

### 6-3. 과거 입상이력 → 급수 역추정

동호인 급수 관행이 성적 기반으로 정형화돼 있어 **(대회규모 × 순위 × 종목)만 알면 급수 구간을 규칙으로 매핑 가능.**

예시 룰 (지역 보정계수 필요):
- 전국규모 대회 입상 / 선수 출신 → 자강권 상단
- 지역 C조 우승 → B조 승급권
- 초심/D 경계 → 입상 경험 유무로 분기

**자기신고 급수와 역추정 급수가 크게 어긋나면(신고↓·이력↑) 샌드배깅 플래그 → 검증 큐.** (난제 2와 직결.)

리스크: 지역별 조편성·승급 기준 제각각(전국 일관 공인기준 부재) → 지역 보정계수 필요. 오래된 이력은 실력 변동으로 오차.

### 6-4. 개인 셀프 임포트 (본인 데이터 = 저리스크)

- 스포넷 본인 전적 화면 캡처 + 협회 경기인 등록증 캡처 + 입상 상장 사진.
- 본인이 자기 계정 화면을 캡처하는 것이라 정보주체 본인 제공 → 리스크 낮음.
- **위조 가능하므로 증빙 종류별 신뢰도 가중치** (무증빙 < 캡처 < 협회등록증).
- **밴드·카페 캡처에 타인 실명 섞이는 문제** → 업로드 단계에서 타인 PII 자동 마스킹/차단 로직 필수(없으면 사용자가 오히려 위반 유발).

### 6-5. 다출처 풀링 + 캘리브레이션

FIDE 2024: 초기 레이팅을 최소 5경기·여러 대회 풀링·바닥 1400으로 산출. 배드민국도 **스포넷+위꾹+배프+자가신고를 풀링**해 seed하되:
- 최소 경기 수 미달 시 "측정 중" 상태 표시.
- **각 출처를 배드민국 좌표계로 매핑하는 캘리브레이션 선행** — 공통 선수 몇 명을 양쪽에서 관측해 오프셋 추정. 단순 풀링은 출처별 스케일·시점 차이로 편향.
- 여러 플랫폼 이력 합산 인정 자체가 "기존 데이터를 배드민국으로 끌어오는" 마이그레이션 유인.

## 7. 배드민국이 지금 당장 만들 것 TOP 5

현재 앱 구조(급수 3축 + MMR RPC) 위에서, 의존성 순서대로:

### 1. 주최자용 CSV 대회결과 임포터
콜드스타트 최대 지렛대. 표준 템플릿 + EUC-KR/UTF-8 자동감지 + 미리보기·중복병합 + 타인 PII 동의 체크. **주최자 1명 업로드 → 참가자 수십 명 자동 온보딩.** (이미지 OCR은 v2로 미뤄도 되지만 CSV는 지금.)

### 2. 급수 → MMR + RD(불확실성) 밴드 매핑
기존 MMR RPC에 **RD(또는 신뢰도) 컬럼 1개 추가.** 급수별 (중앙 MMR, 초기 RD) 밴드 테이블. 바닥값 설정. 이식 급수는 초기엔 순위로만 신뢰. → 난제 1 직접 해결.

### 3. Provisional 큰-K 수렴 엔진 (Glicko-2 스트리밍 변형)
신규 첫 5~10경기 이동폭 2~4배 + 무활동 RD 증가 + 첫경기 스코어마진 역산. TrueSkill은 MS 라이선스 이슈 → **Glicko-2 권장.** 경기 드문 신생 서비스엔 배치 갱신보다 경기 단위 스트리밍 갱신. → 초기값 틀려도 자기교정 + 복귀 샌드배깅 자동 흡수.

### 4. 신뢰도 뱃지 ("측정 중 / 검증완료")
MMR 옆에 신뢰도. **단 초보자 UI 원칙상 숫자 2개(MMR+RD)는 혼란** → "실력 측정 중 · 신뢰도 30%" 또는 "검증중/검증완료 뱃지"로 단순화. 신뢰도 낮은 선수는 조편성 시 넓은 밴드 + 주최자 검토 표시. → 샌드배깅 억제 + 초보 안심 동시.

### 5. 자기신고 + 증빙 온보딩 설문
"본인 급수?" 대신 **예측 변수(선수부 여부/구력/클럽/최근 출전조·성적/증빙 캡처).** 답을 "범위"로 변환해 초기 MMR·RD 매핑. 증빙은 선택+보상형(배지/신뢰도↑). 자기신고 ↔ 입상이력 교차검증으로 2급 이상 괴리 시 자동 플래그.

### 실행 우선순위 한 줄 요약

**#1(임포터)로 데이터를 회수하고 → #2·#3(밴드+엔진)으로 MMR을 세우고 → #4·#5(뱃지+온보딩)로 신뢰를 노출한다. 이 다섯 개를 "운영자가 닿는 한 지역 클럽 2~3개"에 100% 채워 국소 유동성을 만든 뒤 복제하라. 넓게 얕게 깔면 어디서도 급수 신뢰가 임계에 못 미쳐 죽는다.**

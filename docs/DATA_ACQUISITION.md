# 배드민국 데이터 획득 실행안 — 스포넷·위꾹·배프에서 가져오기

> 정리 원칙: **밝혀진 것(확인)** 과 **추정(미확인)** 을 문장마다 구분한다. 마케팅 톤 없이, 실행 가능한 것만 남긴다. 자동 크롤링 배제(형사 무죄여도 부정경쟁방지법 (파)목 민사·최대 배상 리스크)는 이미 확립된 전제로 유지한다.

---

## 1. 핵심 결론

**세 플랫폼 모두, 현실적 획득 경로는 딱 두 개다.**

1. **본인 위임(셀프 온보딩)** — 선수가 자기 계정 화면을 **직접 캡처/복사**해 배드민국에 제출. → 확실히 합법, 마찰 낮음, **커버리지는 느림**.
2. **주최자 협조(대회 단위 이관)** — 대회를 연 협회·클럽이 **자기 대회 결과 원본**을 배드민국 CSV 임포터로 넘김. → 확실히 합법, **커버리지 효율 최고(1건당 수십~수백명)**, 대신 제휴 선행 필요.

**자동 수집(봇 크롤링·사용자 계정 위임 스크래핑)은 세 플랫폼 모두 배제**한다. 크롤링은 (파)목 민사 리스크, 사용자 계정 위임 스크래핑은 "그 순간 회원이 되어 회원 약관의 자동접근 금지가 적용"되는 함정 때문에 오히려 비회원 페이지 파싱보다 지위가 불리하다.

**벤더(운영사) 직접 API 제휴**는 셋 다 **공개 API·개발자 문서가 확인되지 않았다(미확인/없을 가능성)**. 중장기 타진 대상일 뿐, 단기 실행 계획에 넣으면 안 된다.

> 한 줄 요약: **"봇이 가져오는 게 아니라, 사람(선수 본인) 또는 주최자가 넘긴다."** 이 두 축에 자원을 몰고, 나머지는 로드맵에만 둔다.

## 2. 플랫폼별 획득 경로

세 플랫폼의 구조는 놀랍도록 비슷하다: **본인 마이페이지 캡처 + 주최자 관리도구 이관**이 공통 정답이고, **export(CSV/엑셀) 기능 존재 여부가 셋 다 미확인**이라는 것도 공통 병목이다.

### 스포넷 (운영사 ㈜아워스포츠네이션)

| 항목 | 상태 | 내용 |
|---|---|---|
| 급수 체계 | **확인** | 자강 / A / B / C / D / 초심. 시군구 > 시도 > 전국 단위로 급수·국가랭킹 노출 |
| 본인 조회 | **확인** | 로그인 후 '나의 대회'에서 본인 경기 조회, 프로필에 내 급수 표시 |
| 선수조회(타인) | **확인** | 이름으로 상대 전적·입상경력 검색 가능 — 단 **봇 대량조회는 크롤링(배제)** |
| 앱/웹 | **확인** | 안드로이드만(com.osn.sponet). 아이폰은 웹 sponet.co.kr에서 캡처 |
| export 버튼 | **미확인(없을 가능성)** | 사용자용 결과 내보내기 기능 공개 문서에 언급 없음 |
| 캡처 차단(FLAG_SECURE) | **추정(차단 없음)** | 일반 스크린샷 가능해 보임 — 실기기 확인 필요 |
| 운영사 정황 | **추정** | 'SPONET 종료' 안내 페이지 존재, 연락처 노출(hykim@oursponet.com / 010-2768-7656). 사업 축소 국면이면 **데이터 이관 협상 기회**일 수 있음(양날: 인력 이탈로 무응답 가능성도) |

- **본인이 내줄 수 있는 것**: 내 급수 스크린샷 1장(→ 즉시 검증), 나의 대회 전적/입상 캡처.
- **배드민국 수신**: 온보딩 증빙 캡처 업로더(구현됨) → OCR+수동검수 → claim. **스포넷 급수(자강~초심) → 배드민국 MMR 매핑 테이블 필요**.

### 위꾹라이브

| 항목 | 상태 | 내용 |
|---|---|---|
| 공개 URL | **확인** | 대회 안내 `wekkuklive.com/apply_contest/{ID}`, 대진 데모 `/matchdemo/demo`, 요강 PDF가 인증 없이 열림(`file/contest_badminton_file/...`) |
| 결과·점수 완전공개 여부 | **미확인(추정)** | 페이지가 SPA(JS 렌더)라 최종 대진표·경기별 점수가 비로그인 완전공개인지 원문 확인 실패. 대회별로 공개 범위 다를 수 있음 |
| 주최자 도구 | **확인** | `order.wekkuklive.com`(토너먼트 관리) + 심판 앱(com.wekkuk.referee). 심판이 입력한 점수가 서버 자동 저장 → **주최자가 참가자·대진·점수·순위 원본 소유자** |
| 참가자 앱 | **확인** | 개인 경기 통계·실시간 대진표·라이브 영상 제공 → 본인 성적 캡처 가능. VOD는 유튜브 공개 |
| export(CSV/엑셀) | **미확인(추정)** | 명단 다운로드는 있을 가능성 높으나 결과·순위 export 여부 불명 |

- **본인이 내줄 수 있는 것**: 참가자 앱의 내 경기기록/통계 캡처.
- **주최자가 내줄 수 있는 것**: 관리자 화면의 참가자 명단·대진·결과(export 없으면 캡처/인쇄물).
- **배드민국 수신**: 주최자분 → CSV 임포터, 본인분 → 캡처 업로더 → imported_players claim.

### 배프 (Badminton Friends)

| 항목 | 상태 | 내용 |
|---|---|---|
| 마이페이지 | **확인**(앱스토어 설명) | 내 전적, 대회 참가내역, 프로필·급수 표시 |
| BFR(배프 레이팅) | **확인** | Elo 기반 개인 레이팅 산정·표시 → **배드민국 MMR과 동계열 지표, 시드로 최적** |
| BFR 신뢰도 | **확인(주의)** | 배프 스스로 "베타·초기점수·K값 조정 중"이라 명시 → **절대값 신뢰 금지, 넓은 신뢰구간 시드로만** |
| 급수 표기 | **확인** | S/SA/A/B/C/D/E/F 등 |
| 주최자 도구 | **확인** | 대진표 자동작성·실시간 점수판·입금/상품/운영 관리 메뉴 |
| 대회 웹페이지 | **확인** | badmintonfriends.co.kr 노션형 URL, 비로그인에 요강 공개. **단 개인별 성적·대진 결과의 웹 완전공개 여부는 미확인** |
| export | **미확인(추정: 미제공 가능성)** | 공개 문서에 내보내기 명시 없음. 카톡/인스타 채널로 직접 문의 필요 |

- **본인이 내줄 수 있는 것**: 마이페이지 전적/급수 캡처 + **BFR 점수 캡처**(질 좋은 시드) + 대회 참가내역.
- **배드민국 수신**: 캡처 업로더 → OCR → **BFR 구간 → 배드민국 MMR 구간 매핑 테이블**, 급수는 BFR 결측 시 대체 시드.

> **세 플랫폼 공통 병목(반드시 인지)**: ① 결과의 비로그인 완전공개 여부 미확인(위꾹·배프), ② export 기능 부재 가능성(셋 다) → 부재 시 주최자 **수작업 재입력 or 캡처 파싱** 부담 증가, ③ 공개 API 전무. → 그래서 **"본인 캡처 + 주최자 문서"** 외의 자동 경로에 기대면 안 된다.

## 3. 법적 안전지대 — 크롤링과 무엇이 다른가

### 안전한 이유 (본인 제공 경로)

- **개인정보보호법 의무는 '개인정보처리자'에게 부과**된다. **정보주체 본인이 자기 정보를 열람·복제해 원하는 곳에 제공하는 행위 자체를 금지·처벌하는 조항은 없다.** 본인의 자기결정권 행사이므로 적법. 배드민국은 받은 뒤 **본인 동의(제15·17조) 근거로 처리**하면 된다.
- **사실 데이터는 저작물이 아니다.** 급수 숫자·승패 기록은 창작성이 없어 저작권 보호 대상이 아니다(선택·배열에 창작성 있어야 편집저작물).
- **DB제작자 권리(저작권법 제93조)가 보호하는 것은 'DB의 전부 또는 상당한 부분'의 복제.** 본인 기록 한 줄·개별 소재는 상당한 부분에 못 미쳐 원칙적으로 침해 아님.

### 크롤링과 다른 지점

- 크롤링은 **배드민국 서버/봇이 대상 서버에 반복 접근**해 DB를 재구성 → (파)목 성과도용(민사) + 제93조 제2항 "반복·체계적 개별소재 수집 = 상당한 부분 복제 간주" 위험.
- 본인 캡처/붙여넣기는 **자동 접속·서버 부하·접근권한 쟁점이 원천적으로 발생하지 않는다.** 야놀자-여기어때 판례(대법원 2021도1533)의 접근권한 논쟁 자체가 성립 안 함.
- **형사 무죄 ≠ 안전**: 같은 여기어때 사건에서 **민사는 부정경쟁방지법 성과도용으로 배상 인정**. 형사·민사 법리는 완전 별개. → COLDSTART의 크롤링 배제 결정은 이 이원 리스크에 정확히 부합.

### 보조 법적 수단 (근거는 확실, 확산은 느림)

- **개인정보 열람·사본 요구권(제35조)**: 규모 무관 모든 처리자에 적용, **10일 내 사본 제공 의무**(수수료·우송료 청구 가능). 단 "본인에게" 제공을 강제할 뿐 → **본인이 받아 배드민국에 재업로드하는 2단계**. 처리자가 요약·일부만 줄 수 있고 지연·수수료 리스크. **캡처가 더 빠를 수 있어 보조 수단**.
- **로앤비 판례(대법원 2014다235080) 이익형량**: 이미 공개된 개인정보는 "동의가 객관적으로 인정되는 범위 내"면 별도 동의 없이 처리 가능. **단 이는 형사 논리에 가깝고 (파)목 민사 리스크는 별개** → "공개정보라 크롤링해도 된다"로 확대 금지. **본인 제공/제휴로 들어온 공개기록의 처리 정당화 근거로만** 사용.

### 타인 PII 섞임 처리 (필수 안전장치)

- 본인 대진 캡처에 상대 실명이 함께 찍히면 → **업로드 파이프라인에서 타인 이름 자동 마스킹(홍길동→홍○○)** 하거나, 저장 필드를 '본인 승패·급수'로 한정하고 상대 식별정보는 저장하지 않는다.
- **"문서에 이미 적혀 있다"는 사정만으로 타인 정보 전파가 면책되지 않는다**(제59조 누설 관련 판례 존재). 상대는 **익명 해시 또는 가입 후 매칭**으로 보완.
- 리스크 대비 비용이 낮으므로 **선택이 아니라 필수**.

### 전송요구권(마이데이터) — 현재 불가 (정직한 한계)

- 2025 개정법 전송요구권은 **전송의무자 지정 요건**(대체로 평균매출 1,800억 초과 + 정보주체 100만명 이상, 또는 민감·고유식별 5만명 이상 수준)이 있고 초기 적용은 의료·통신·에너지 중심.
- 스포넷·위꾹·배프 같은 신생/중소 배드민턴 앱은 **이 규모 요건에 미달할 개연성이 매우 높다(각 사 실제 매출·규모 미확인·추정)**. → **마이데이터로 강제 수신은 현시점 사실상 불가.** 로드맵 항목으로만 보관.

## 4. 기술 실행 — 정확도와 인간검증 루프

핵심 원칙: **어떤 자동 파서도 "확정"이 아니라 "초안"이다. 최종 확정은 선수 본인 검수를 강제한다.**

### (A) 캡처 이미지 OCR — 주력 후처리

- **도구**: 네이버 CLOVA OCR(표 구조 보존 Table 템플릿) 또는 Google Cloud Vision `DOCUMENT_TEXT_DETECTION`. **Tesseract는 표/저화질에 약해 1차 도구로 부적합.**
- **정확도(정직하게)**:
  - 벤더 마케팅의 "97~99%"는 **그대로 신뢰 금지(추정·미검증)**. 독립 비교의 "한글 인쇄체 85~90%대"도 엄밀한 벤치마크 아님.
  - **확실한 것**: 손글씨·저해상도·스크롤 캡처·다크모드 스샷에서 정확도 급락(손글씨 Tesseract 20~40% 붕괴, 클라우드도 자릿수 경계 오류 잦음).
- **치명적 함정**: 급수·점수에서 `0↔O, 1↔l/I, 5↔6, 8↔B` 혼동 → **숫자 칼럼은 정규식 형식검증(급수 = A~D조 + 정수) 후 "이 값이 맞나요?" 재확인 필수**.
- **루프**: OCR confidence 임계값 → 낮으면 자동채택 금지, 사용자 확인 UI로.

### (B) 주최자 문서 파싱 — 커버리지 효율 최고

- **네이티브 텍스트 PDF/엑셀**: `pdfplumber`·`camelot`·`tabula`로 표를 무손실 추출 → **신뢰도 매우 높음(실질 확정 가능)**.
- **HWP/HWPX**: 한글 문서 파서로 표 텍스트화.
- **스캔 이미지 PDF**: OCR 경유 → 위 (A)의 한계 그대로 승계.
- **브래킷(대진표 그림)**: 시각 레이아웃 복잡 → 단순 표추출로 승부관계 깨짐. **표 형태 '경기 결과 리스트'를 우선 요청**, 브래킷은 보조.
- **실무 병목은 파싱보다 정규화·동일인 판정**: 주최자마다 양식 제각각 → **(이름+소속클럽+지역) 다중키 또는 claim 연결**. 동명이인 이름만으론 충돌.

### (C) 텍스트 붙여넣기 파싱 — OCR보다 우수하나 조건부

- 2단계: (1) 정규식으로 뻔한 패턴(이름·급수코드·'n승 m패'·날짜) 1차 파싱, (2) 흐트러진 잔여분만 LLM에 **JSON 스키마 강제**.
- **LLM 필수 안전장치 = grounding 검증**: 추출값이 실제 원문 문자열에 존재하는지 **substring 대조, 없으면 폐기(환각 방지)**.
- **한계**: 대부분 앱이 이미지/캔버스 렌더라 **복사할 텍스트가 없으면 이 경로 불가 → 캡처 OCR로 회귀**.

### 방법별 요약

| 방법 | 정확도 | 마찰 | 비고 |
|---|---|---|---|
| 텍스트 PDF/엑셀 파싱 | 매우 높음 | 낮음(문서 확보 시) | **주최자 경로의 정공법** |
| 텍스트 붙여넣기 | 높음(OCR 오인식 없음) | 중(복사 가능한 텍스트 있어야) | 앱이 캔버스면 불가 |
| 캡처 OCR | 중(조건 의존) | 중 | **반드시 본인 검수 강제** |
| 스캔 이미지 PDF | 낮음~중 | 중 | OCR 한계 승계 |

## 5. 배드민국이 지금 만들 것 (기존 자산 위에 추가)

이미 있음: **주최자 CSV 임포터 / 온보딩 증빙 캡처 업로드 / imported_players + claim**. 위에 얹을 구체 기능:

1. **급수·레이팅 매핑 테이블 + 관리 UI** *(가장 시급)*
   - 스포넷(자강/A/B/C/D/초심), 배프(S~F 급수 + **BFR 구간**), 위꾹 급수 → **배드민국 MMR 구간**으로 환산하는 룩업 테이블.
   - 시드는 **넓은 신뢰구간 + 시간감쇠 가중치**로 넣고, 배드민국 실경기 결과로 Elo 교정. **BFR은 확정등급 취급 금지(배프도 베타 명시)**.

2. **캡처 → OCR 반자동 파서 + 본인 확인 UI**
   - CLOVA/Vision 연동, confidence 임계값 필터, **숫자 칼럼 정규식 검증**, "이 값이 맞나요?" 확정 화면. OCR은 초안, 확정은 사람.

3. **타인 PII 자동 마스킹 파이프라인** *(법적 필수)*
   - 업로드 이미지/텍스트에서 상대 실명 자동 마스킹, 저장 필드 화이트리스트('본인 급수·승패'만), 상대는 익명 해시. **선택 아님.**

4. **주최자 파트너 온보딩 키트 + 문서 임포터 포맷 확장**
   - CSV 외 **텍스트 PD F·엑셀·HWP 직접 업로드** 지원, 열 매핑 반자동 UI(템플릿 몇 개 + 나머지 수동 매핑).
   - 파트너 키트에 "**export 기능 있는지 벤더에 문의**" 절차 포함(스포넷/위꾹/배프 각 채널), 없으면 캡처·인쇄물 경로 안내.
   - 주최자 인센티브(무료 운영툴·상금정산 등)로 락인된 주최자 유인 — **콜드스타트 효율 최고 채널이므로 여기에 영업 리소스 집중**.

5. **동일인 판정 + claim 강화**
   - (이름 + 소속클럽 + 지역 + 출생년도) 다중키 매칭. **대한체육회 스포츠지원포털**(공개 검색: 이름·출생년도·성별·시도·소속팀)은 **급수엔 부적합하지만 신원 교차검증 보조**로만 활용(개별 조회만, 대량 자동수집 금지).
   - 업로드 시 "**본인 데이터이며 제3자 제공 동의**" 체크박스 → 동의 근거를 로그로 남김.

## 6. 하지 말 것 (레드라인)

- ❌ **서버·봇으로 스포넷 '선수조회', 위꾹 공개 URL 순회, 배프 대회 페이지를 자동 수집** — (파)목 민사·제93조 반복수집 리스크. 형사 무죄여도 안전 아님.
- ❌ **사용자 계정 자격증명 위임 스크래핑**(봇이 대신 로그인) — 그 순간 '회원'이 되어 **회원 약관의 자동접근 금지가 적용**(비회원 페이지 파싱보다 오히려 불리). + 자격증명 보관 보안 리스크 + 계정정지. 금융권이 스크래핑 금지→API로 몰아간 규제 추세와도 역행.
- ❌ **개별 캡처를 모아 원본 순위 DB를 재구성·재배포** — 개별 소재라도 "반복·체계적"이면 상당한 부분 복제로 간주. **선수 프로필 단위 저장으로 한정**, 순위표 재구성 금지.
- ❌ **미claim 타인 기록의 공개 전시** — imported_players는 **최소정보·비공개 보관**, 본인이 claim한 선수만 프로필 노출. 이익형량이 불리해진다.
- ❌ **BFR·타 플랫폼 레이팅 산식의 무단 역설계·대량 재현** — "수치 자기신고 수용"과 "알고리즘 베낌"은 구분. 후자는 (파)목 소지.
- ❌ **벤더 API 제휴를 단기 실행계획에 편성** — 공개 API·문서 셋 다 미확인, 배프·위꾹은 급수 표준을 노리는 **경쟁·인접 사업자**라 협력 유인 약함. 저비용 타진은 하되 **중장기 옵션**으로만.
- ❌ **마이데이터 전송요구권 기대** — 대상 규모 요건 미달 개연성 높음(추정). 로드맵에만.
- ⚠️ **조문 번호·징벌배수(3배/5배) 확정 인용 주의** — 부정경쟁방지법 성과도용 조항은 개정으로 (카)목→(파)목 이동 이력. **최종 문안은 법무 확인 후 확정**(현재는 미확정 표시 유지).
